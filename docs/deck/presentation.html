The Problem

Training humanoid robots burns cloud money — silently

Reinforcement-learning skills (walking, grasping) for robots like the Unitree G1 are compute-heavy and expensive. Three failure modes quietly drain budgets:

  • Idle GPU spend — H100 / MI300X instances at $3–5+/hr left running before & after training.
  • Silent divergence / collapse — RL runs diverge or flatline for hours before anyone notices.
  • No audit & provenance — no record ties a finished checkpoint to its exact GPU cost, metrics, and approvals.

Managing async sim containers, live telemetry, and cloud budgets is a manual, error-prone operational headache.

The Solution

Governance for ML infra — as a BPMN process

A governed, agentic, human-in-the-loop orchestration on UiPath Maestro BPMN that runs the whole training lifecycle: AI agents do the analysis; humans approve the critical cost and physics-safety gates.

Agentic AI

UiPath Agent Builder agents classify each training anomaly, score risk, and decide fast-track vs. human review.

RPA

A UiPath RPA Robot provisions cost-optimal GPUs (AWS/GCP/AMD), SSH-kills bad runs, and writes to cloud consoles with no clean API.

Action Center (HITL)

Researchers approve checkpoints & high-cost actions on rich, read-only AI context.

Active cost containment

Diverging runs are terminated instantly — saving thousands in wasted GPU hours.

How it works

From training request to audited checkpoint

1 · INGEST
Researcher requests a skill (e.g. walk-gait-v2) via a form / GitHub.
2 · BROKER
RPA finds the cheapest available GPU across AWS / GCP / AMD.
3 · DEPLOY
SSH-launches the MuJoCo RL container on the chosen instance.
4 · MONITOR
AI agent watches reward/loss; on collapse → instant SSH shutdown + alert.
5 · HITL
Healthy run → Action Center card; researcher approves the checkpoint.
6 · AUDIT
Register weights, tear down GPU, log immutable cost & approval trail.

An agentic Fast-Track gateway auto-resolves low-risk anomalies (e.g. release an idle instance); high-risk / production-bound ones escalate to a human.

UiPath Components & Agent Type

Built on the full UiPath stack

  • UiPath Maestro (BPMN 2.0) — the orchestration core (3 processes / 8 containers / 9 roles).
  • UiPath Agent Builder — Telemetry Analyst, Anomaly Risk, Cost & Impact, compliance-summary agents.
  • UiPath Action Center — human-in-the-loop Researcher Review Board (registered action type + forms).
  • UiPath RPA Robot + API Workflows — GPU provisioning & cloud/registry write-back (QueueProcessor.xaml).
  • UiPath Orchestrator — Queues (IncidentReports), Folders, Jobs, transaction status.

Agent Type — both

Low-code: UiPath Agent Builder agents orchestrated by Maestro.   Coded: the same contracts implemented as deterministic, testable Python task nodes (Claude primary / Gemini fallback).
